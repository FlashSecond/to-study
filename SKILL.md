# to-study — 团队学习协作（v6.2 精简版）

## 概述

虚拟学习内容制作团队。接收任何资料 → 七阶段流水线 → 专业教学级教材。

**设计哲学**：先动手后理解 / 三列组织（一目了然+快速查找）/ 任务三级递进（基础→组合→综合）/ 排障体系独立

---

## 核心红线

不达标直接打回：

| # | 红线 | 阈值 |
|:--:|------|:--:|
| 1 | 8 模块完整 | 缺失任何模块 → 打回 |
| 2 | 字数达标 | framework ≥30000字/章；tool/theory ≥20000字/章 |
| 3 | 代码/示例充分 | framework/tool ≥5 可运行示例；theory ≥5 论证示例 |
| 4 | Mermaid 图 | 每章 ≥3 张 |
| 5 | 功能矩阵 | 格式：功能分类/签名/参数/★常用度/使用前提/关联功能 |
| 6 | 结构-功能映射 | 文本标注图 + Mermaid 映射图 |
| 7 | 陷阱+排障决策树 | ≥5 陷阱（❌/✅+现象+根因）+ Mermaid 决策树 |
| 8 | 练习含答案 | ≥6 题（基础≥3 + 进阶≥2 + 挑战≥1），答案折叠 |
| 9 | 全局速查中枢 | 阶段6：快捷键+术语对照+标签云+同义词映射+搜索索引 |

## 统一 8 模块模板（强制）

> 模块顺序不可更改。每个模块按"可扫读表格 + 可检索入口"双重组织。

---

### 模块 1：5 分钟极速上手卡

零门槛首个可感知成果。

| 子项 | 要求 | 类型条件 |
|------|------|:--:|
| 环境与安装 | 按 OS 分，复制即用的命令。附带排错指引→模块7 | framework/tool |
| 启动检查 | 复选框验证：环境✅/版本✅/依赖✅ | framework/tool |
| 首次体验 | framework/tool：最小闭环 ≤3行/步。theory：3-5核心问题+阅读路径 | 全部 |
| 成功后指引 | 一行话：→模块3理解原理 / →模块4做任务 | 全部 |

---

### 模块 2：结构全景图

先见森林再见树木。

| 子项 | 要求 |
|------|------|
| 一句话定义 | ≤30字：`{名称} 是一个 {定位}，解决 {核心问题}` |
| 核心价值 | 3-5 条：为什么存在？不用会怎样？与替代的本质区别 |
| 全景导航图 | Mermaid `graph TD`（framework/tool：模块关系图；theory：概念层级图） |
| 快速定位 | "我想做 X → 模块 Y / 功能 Z" 对照表，高频置顶 |
| 同类对比 | 名称/定位/优势/劣势/场景 对照表 |

---

### 模块 3：核心概念与架构

从"会抄"到"理解"。

| 子项 | 要求 |
|------|------|
| 术语表 | ≤20 术语（中英/一句话/跳转）；theory 术语表=全文概念索引 |
| 结构模型 | Mermaid `classDiagram` 或 `entity-relation`；theory 为概念模型层级 |
| 概念详解 | ≥3 个，每个 ≥600 字（定义+设计意图+示例+辨析+边界）。≥3 示例合计 ≥60 行代码。辨析表 ≥3 行 |
| 底层机制 | Mermaid `sequenceDiagram`（≥8 参与者）+ 每步 ≥80 字解析；theory 展示论证逻辑链 |
| 设计理念 | 3-5 句话（如"一切皆文件""声明式配置"） |
| 结构-功能映射 | **必含**：文本标注图 + Mermaid 映射图 |

---

### 模块 4：任务导向操作指南

按"我想做……"组织。每项一张卡片，三级递进。

| 子项 | 要求 |
|------|------|
| 任务卡片 | 含：目标/结果预览 → 步骤流程（Mermaid flowchart ≥5节点）→ 每步操作（合计≥30行代码，含决策注解）→ 耗时 → 前置标签。**每卡 ≥800字** |
| 三级递进 | 基础≥2（⭐，≥500字）→ 组合≥2（⭐⭐，≥800字）→ 综合≥1（⭐⭐⭐，≥1500字） |
| 频次排序 | 高频置顶 |
| 互相引用 | 每卡底部标注 →模块5功能参考 / →模块3概念 / →模块7排障 |

> 按耗时（3/10/30分钟+）、难度（⭐/⭐⭐/⭐⭐⭐）筛选

---

### 模块 5：功能参考手册

API/命令/模型的详尽字典。随用随查。

| 子项 | 要求 |
|------|------|
| 功能矩阵 | 表格：功能分类 / 签名 / 参数（名·类型·默认·说明）/ ★常用度 / 最小示例 |
| 条目扩展 | 描述 + 使用前提与限制 + 关联功能（交叉引用）+ 最简可运行示例 |
| 速查子表 | 所有参数/配置默认值一览；仅 framework/tool |

> 三重索引：按模块浏览 / 按功能 A-Z / 按参数名搜索

---

### 模块 6：实战案例库

端到端、可复用的工作流模板。按行业/岗位组织（与模块4按功能组织互补）。

| 子项 | 要求 |
|------|------|
| 标题格式 | `{行业/岗位} - {任务名}` |
| 6 要素 | ①场景(≥100字) ②前置条件 ③分步操作(≥40行代码，标注关键决策+为什么) ④决策解说(≥300字，≥3决策点) ⑤成果物+输出样例 ⑥可复用模板。**每案例 ≥2000字，每章 ≥2案例** |
| 规模递进 | 单功能 → 模块组合 → 完整项目 |

---

### 模块 7：排障诊所

陷阱对照 + FAQ + 排障决策树 + 错误码词典。

| 子项 | 要求 | 类型条件 |
|------|------|:--:|
| 陷阱对照 | ≥5 个，每个 ≥300 字：❌错误(≥3行) + ✅正确(≥3行) + 现象(≥50字) + 根因(≥80字) + 避免(≥50字) | 全部 |
| FAQ | ≥5 条，每条 ≥150 字。诊断步骤+方案，标注"最常见于：{场景}" | 全部 |
| 决策树 | Mermaid `flowchart TD`：入口现象→逐级症状→原因→修复 | 全部 |
| 错误码词典 | 错误码/日志字段解释；theory 替换为逻辑谬误类型 | 全部 |

> 支持按报错字符串全文检索

---

### 模块 8：练习与自测

| 子项 | 要求 | 类型条件 |
|------|------|:--:|
| 基础理解 | ≥3 题 ⭐，选择/填空/简答 | 全部 |
| 深度分析 | ≥2 题 ⭐⭐，代码分析/论证分析 | 全部 |
| 动手实验 | ≥1 题 ⭐⭐⭐，给目标不给步骤 | 全部 |
| 参考答案 | 折叠（`<details>`），含思路+验证+延伸。基础答案≥150字，挑战≥300字 | 全部 |

### 全局速查中枢（阶段 6 独立文件）

| 工具 | 内容 | 条件 |
|------|------|:--:|

| 命令行速查 | 常用命令+参数一览 | framework/tool |

| 配置默认值 | 配置项+默认值+可改范围 | framework/tool |

## 七阶段工作流

> **强制逐阶段执行，禁止跳阶段。禁止凭记忆替代采集。阶段间不中断等确认。**

### 阶段 1：内容采集 `ts-collector`

⚠️ 必须脚本采集，禁止 AI 直接读取后复述。

策略：源书文件读取 / `web_fetch` / PDF / EPUB / DOCX / MD/TXT/HTML

### 阶段 2：结构规划 `ts-structurer`

类型判定（framework/tool/theory）→ 章节拆分 → 写入 `00_结构规划方案.md`

### 阶段 3：知识提炼 `ts-extractor`

从源内容提取以下素材，写入 `_提炼清单.md`：

| 强制素材 | 推荐素材 |
|------|------|
| 一句话定义 / 核心价值 / 术语 / 概念辨析 / 底层机制 / 陷阱(≥5) / 排障素材 / 适用边界 | 性能要点 |
| 结构映射素材 / 功能矩阵素材 / 任务卡片素材 / 案例素材 | 高级技巧 / 最佳实践 |

### 阶段 4：内容编撰 `ts-writer`

严格按「统一 8 模块模板」编撰，每章一个 `.md`。

#### 🚨 子 Agent 深度编撰规范

> 一章一 Agent，深度优先。主 Agent 只负责派遣、验证、断点。

| 规则 | 说明 |
|------|------|
| 一章一 Agent | 不打包多章 |
| 模板注入 | task 中必含本文档全部 8 模块定义 + 深度标尺 |
| 章节素材注入 | task 中注入该章的阶段 3 提炼素材（术语/概念/API/陷阱/案例要点） |
| 深度基线 | task 首尾各重复：「本章 ≥30000字(framework)/≥20000字(tool/theory)，8模块完整，达到深度标尺。不是填空，是深度创作。」 |
| 并行批次 | 每批 3-4 Agent 并行，等待全部完成后验证再下批 |
| 产出验证 | ①≥8模块 ②字数≥红线 ③概念≥3/案例≥2/陷阱≥5/练习≥6 ④不合格→重写 |
| 写入 | 子 Agent 直接用 `write` 写 `.md` 终稿路径 |
| 断点 | 每章完成后更新 `_checkpoint.json` |

**降级方案**：`sessions_spawn` 连续 3 次 401/402/403 → 主 Agent 直写，同深度基线，每批仍只写 1-2 章。

### 阶段 5：排版设计 `ts-designer`

视觉提示：💡技巧 ⚠️注意 ❌错误 ✅正确 🤔思考 📌要点 🔗延伸 🔀辨析

检查：一目了然栏全部表格/列表/步骤编号；快速查找栏给出索引词/搜索键/跳转链接。

### 阶段 6：全局速查中枢 `ts-designer`

写入 `_速查中枢.md`：快捷键表 / 术语中英对照 / 标签云 / 同义词映射 / 搜索索引

### 阶段 7：质量审核 `ts-reviewer`

**十二维审核**：

| 维度(权重) | | 维度(权重) | |
|------|------|------|------|
| 📐 内容深度(16%) | 💻 代码质量(16%) | 🏗️ 结构完整(13%) | 📊 图表丰富(10%) |
| 📋 功能矩阵(8%) | 🗺️ 结构映射(8%) | 🔍 功能覆盖(8%) | 🛡️ 陷阱排障(8%) |
| 📝 练习质量(6%) | 🎯 教学示例(4%) | 🇨🇳 中文表达(2%) | 🔬 技术准确性(1%) |

**打回条件**：
- 8 模块缺失 / 功能矩阵缺失 / 结构映射缺失 → 直接打回
- 内容深度/代码/陷阱/功能矩阵任一项 <80 → 条件放行（必改项）
- 练习无答案 / 缺速查中枢 → D 级

| 等级 | 分数 | 处理 |
|:---:|:---:|------|
| A | ≥90 | ✅ 放行 |
| B | ≥80 | ✅ 放行+建议 |
| C | ≥70 | ⚠️ 条件放行+必改清单 |
| D | <70 | ❌ 打回重写 |

审核报告写入 `XX_审核总报告.md`。
